No, you do not typically create the RPA (Release Pipeline Artifact) manually. In Konflux, the RPA is automatically generated as part of the build and release pipeline process. The RPA is created by the Konflux CI/CD system when you:

1. Push your code to the repository
2. Configure your application in the Konflux UI
3. Set up your build and release pipeline

The RPA is automatically generated during the build process, capturing:
- The application's source code
- Build artifacts
- Metadata about the build
- Release information

To get an RPA created, you typically need to:
- Have a valid source code repository
- Configure your application in Konflux
- Set up your build pipeline
- Trigger a build (either manually or through a commit)

If you want to verify or track RPAs, you would do so through the Konflux dashboard or CLI, rather than manually creating them.
